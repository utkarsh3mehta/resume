Solution 1:

Task:

Get data from 700 servers in Azure cloud inside a closed network.

Problem statement:

We generally use Invoke-AzureRmVMCommand to connect to our cloud infrastructure servers and run scripts locally. But this wouldn"t work if we tried to "GET" any data from the servers locally.

Solution:

I created a small Flask application and hosted it, once on a web server and once on Azure web app. The application simply takes the data from the GET URL and saves it to a CSV file. Then, using the above-mentioned cmdlet, I ran a script locally that collects the data and invokes a web request to my web API. This way I got data from our entire environment in just a matter of hours.

How things would have happened without the solution?

Resources would have been dedicated to this task and trained as it required some knowledge on Registry Entry. Also, because the server count is high, it would have taken at least 3 days to completely get the information. And not to ignore, human error, such as wrong or missing information.
